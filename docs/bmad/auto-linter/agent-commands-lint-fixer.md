# Agent Commands and Capabilities: lint-fixer

## Core Capabilities Identified

### 1. Run All Lints (Primary Command)
- **Trigger:** `fix`
- **Function:** Run both backend and frontend linters with auto-fix
- **Behavior:**
  - Context detection (weave-mobile/, weave-api/, or project root)
  - Execute backend linters: `uv run ruff check .` + `uv run ruff format .`
  - Execute frontend linters: `npx tsc --noEmit` + `npm run lint --fix`
  - Iterate up to 3 times until clean or max iterations reached
  - Display summary results with counts and remaining issues

### 2. Frontend Only
- **Trigger:** `frontend`
- **Function:** Run frontend linters only (TypeScript + ESLint)
- **Behavior:**
  - Execute: `npx tsc --noEmit` + `npm run lint --fix`
  - Iterate up to 3 times
  - Summary results: fixed count, remaining issues with file:line details

### 3. Backend Only
- **Trigger:** `backend`
- **Function:** Run backend linters only (Ruff)
- **Behavior:**
  - Execute: `uv run ruff check .` + `uv run ruff format .`
  - Iterate up to 3 times
  - Summary results: fixed count, remaining issues with file:line details

### 4. Setup Pre-commit Hook
- **Trigger:** `setup-hook`
- **Function:** Install pre-commit hook for automatic linting
- **Behavior:**
  - Create `.git/hooks/pre-commit` script
  - Configure to run lint-fixer before commits
  - Automatically offered on first run
  - Provides escape hatch documentation (--no-verify)

## Command Structure (YAML)

## Prompt Template Structure

### #fix-all-lints
**Purpose:** Main command - run all linters with context awareness

**Process:**
1. Detect current working directory context
2. If in weave-mobile/: run frontend lints
3. If in weave-api/: run backend lints
4. If in project root: run both backend and frontend
5. Iterate up to 3 times per component
6. Collect results and generate summary report
7. Return exit code (0=success, 1=remaining issues)

**Output Format:**
```
✅ Backend: 23 issues fixed, 0 remaining
⚠️  Frontend: 15 issues fixed, 3 remaining (max iterations reached)

Remaining issues:
  weave-mobile/src/components/ImageGallery.tsx:45 - Type error
  weave-mobile/src/hooks/useImageList.ts:12 - Missing return type
  weave-mobile/app/(tabs)/index.tsx:89 - Unused variable

Exit code: 1
```

### #fix-frontend-lints
**Purpose:** Run only frontend linters

**Process:**
1. Check if in weave-mobile/ or has package.json
2. Run `npx tsc --noEmit` (capture errors)
3. Run `npm run lint --fix` (apply fixes)
4. Iterate up to 3 times
5. Report summary with file:line details for remaining issues

### #fix-backend-lints
**Purpose:** Run only backend linters

**Process:**
1. Check if in weave-api/ or has pyproject.toml
2. Run `uv run ruff check . --fix` (apply fixes)
3. Run `uv run ruff format .` (format code)
4. Iterate up to 3 times
5. Report summary with file:line details for remaining issues

### #setup-precommit-hook
**Purpose:** Install git pre-commit hook

**Process:**
1. Check if .git/ directory exists
2. Create .git/hooks/pre-commit with executable permissions
3. Write script that runs lint-fixer
4. Configure to block commit if linting fails (exit 1)
5. Document escape hatch: `git commit --no-verify`
6. Confirm successful installation

**Hook Script Content:**
```bash
#!/bin/bash
# Auto-generated by lint-fixer agent
# Run linting before commit

echo "Running lint-fixer pre-commit hook..."
lint-fixer fix || exit 1
echo "✅ All linting passed!"
```

## Workflow Integration Plan

**No external workflows needed** - All functionality is self-contained within prompt templates using Bash tool for command execution.

**Architecture:** Simple Agent with inline prompts (no workflow references)

## Advanced Features

### Context Detection Logic
- Check `pwd` output for directory names
- Verify presence of key files (package.json, pyproject.toml)
- Adapt behavior based on detected context

### Iteration Tracking
- Count iterations per linter run
- Stop at 3 iterations max
- Track what changed between iterations (detect convergence)

### Exit Code Strategy
- 0: All linters passed with zero errors
- 1: Remaining issues after max iterations (report to user)

### First-Run Detection
- Check if `.git/hooks/pre-commit` exists
- If not, prompt user to run `setup-hook`
- Store flag in agent memory (not applicable for Simple Agent)
- Alternative: Always mention hook availability in activation message

## Implementation Notes

### Architecture-Specific Considerations

**Simple Agent constraints:**
- All logic fits within YAML prompt templates
- No persistent memory between runs
- Each execution is independent (stateless)
- No external workflow dependencies

**Technical Requirements:**
- Access to Bash tool for command execution
- Read/write permissions for .git/hooks/
- Ability to parse linter output (text parsing)
- Context detection via filesystem checks

**Error Handling:**
- Handle missing dependencies (uv, npm, npx)
- Handle non-git repositories gracefully
- Handle permission errors on hook installation
- Provide clear error messages with remediation steps

### Summary Output Design

**Summary Priority:**
1. High-level status (✅ passed, ⚠️ issues remaining)
2. Counts (fixed vs remaining per component)
3. Specific file:line details for remaining issues
4. Exit code for CI integration

**Verbose mode:** Not included in initial version (keeping it simple per user preference)
