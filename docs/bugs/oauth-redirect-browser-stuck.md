# OAuth Redirect Issue: Browser Stuck on Google Screen

## Problem Description

When signing in with Google OAuth:
- ✅ User is successfully created in Supabase
- ✅ OAuth callback completes successfully
- ❌ Browser window stays open on Google's OAuth screen
- ❌ App doesn't navigate away from the login screen

## Root Cause Analysis

The issue occurs when the redirect URI from Supabase doesn't **exactly match** the redirect URI expected by `expo-web-browser`. 

### How OAuth Flow Works:

1. App calls `supabase.auth.signInWithOAuth()` with `redirectTo` parameter
2. Supabase returns an OAuth URL
3. App opens URL in browser via `WebBrowser.openAuthSessionAsync(url, redirectTo)`
4. User completes OAuth on Google
5. Google redirects to Supabase callback
6. Supabase processes OAuth and creates user
7. **Supabase redirects back to app using the `redirectTo` URI**
8. `WebBrowser.openAuthSessionAsync` detects the matching redirect and closes browser
9. App extracts tokens from callback URL and sets session

### The Problem:

If step 7's redirect URI doesn't **exactly match** the `redirectTo` parameter (even by a single character, trailing slash, or case difference), `WebBrowser.openAuthSessionAsync` won't recognize it as a match and will keep the browser open.

However, Supabase has already processed the OAuth and created the user, which is why you see the user in Supabase but the browser stays open.

## Solution

### 1. Verify Redirect URI Configuration

The redirect URI is generated by `makeRedirectUri()` and logged at app startup. Check the console for:

```
╔════════════════════════════════════════════════════════════
║ 🔗 OAUTH REDIRECT URI CONFIGURATION
╠════════════════════════════════════════════════════════════
║ Generated Redirect URI: <YOUR_URI_HERE>
```

### 2. Add to Supabase Dashboard

1. Go to [Supabase Dashboard](https://supabase.com/dashboard)
2. Select your project
3. Navigate to **Authentication → URL Configuration**
4. Add the **exact** redirect URI from the console to "Redirect URLs"
5. **Critical**: Copy it EXACTLY - no trailing slashes, exact case, exact format

### 3. Common Redirect URI Formats

- **Development (Expo Go)**: `exp://192.168.x.x:8081` or `exp://localhost:8081`
- **Production (Custom Scheme)**: `weavelight://`
- **iOS Simulator**: `exp://localhost:8081`

### 4. Verify Matching

After adding to Supabase, the redirect URI must match **exactly**:
- Same scheme (`exp://` vs `weavelight://`)
- Same host (if applicable)
- Same path (usually just `/` or empty)
- **No query parameters** in the base URI (query params are added by Supabase)

## Code Changes Made

### Enhanced Logging

Added comprehensive logging throughout the OAuth flow to help debug:

- Redirect URI generation and comparison
- Browser result details
- Token extraction status
- Session verification
- Navigation state changes

### Timeout Protection

Added a 60-second timeout to detect if the browser gets stuck:

```typescript
const browserPromise = WebBrowser.openAuthSessionAsync(data.url, redirectTo);
const timeoutPromise = new Promise<never>((_, reject) => {
  setTimeout(() => {
    reject(new Error('OAuth browser timeout - browser may be stuck. Check redirect URI configuration.'));
  }, 60000);
});

const result = await Promise.race([browserPromise, timeoutPromise]);
```

### Session Verification

Added explicit session verification after setting tokens to ensure the session was actually created:

```typescript
const { data: { session: verifySession } } = await supabase.auth.getSession();
if (verifySession) {
  console.log('[AUTH_CONTEXT] ✅ Session verified - user is authenticated');
}
```

### Navigation Logging

Added logging to the auth layout to track navigation state:

```typescript
console.log('[AUTH_LAYOUT] 🔍 Auth state check:', {
  hasUser: !!user,
  userId: user?.id,
  inAuthGroup,
  currentSegments: segments,
});
```

2. **Verify Supabase Settings**: Ensure the redirect URI in Supabase matches exactly
3. **Test OAuth Flow**: 
   - Click "Sign in with Google"
   - Complete OAuth
   - Check console for detailed logs
   - Browser should close automatically
   - App should navigate to tabs

## Debugging Steps

If the issue persists:

1. **Check Console Logs**: Look for:
   - `[AUTH_CONTEXT] 🔍 Redirect URI comparison:` - Shows if URIs match
   - `[AUTH_CONTEXT] Browser result received:` - Shows browser response
   - `[AUTH_LAYOUT] 🔍 Auth state check:` - Shows navigation state

2. **Compare Redirect URIs**:
   - Check the logged redirect URI from app startup
   - Compare with what's in Supabase dashboard
   - Ensure they match **exactly** (character by character)

3. **Check for Timeout**: If you see "OAuth browser timeout", the redirect URI definitely doesn't match

4. **Verify Deep Linking**: Ensure your app's deep link scheme is configured correctly in `app.json`
